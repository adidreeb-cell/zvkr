## Обзор датасетов

Проект включает три тестовых CSV-датасета, которые имитируют реальные выгрузки из различных систем учёта студентов университета. Все три датасета намеренно имеют **разные названия колонок** — это ключевая тестовая задача системы: LLM должна автоматически понять, что, например, `"ФИО"`, `"full name"` и `"StudentName"` — это одно и то же поле.

| Файл | Строк | Период | Назначение |
|------|-------|--------|-----------|
| `university_all_years_2016_2020.csv` | ~7 000 | 2016–2020 | Исторические данные (обучение модели) |
| `university_all_years_2020_2024.csv` | ~6 928 | 2020–2024 | Данные среднего периода (валидация) |
| `university_all_years_2025_2030.csv` | ~9 674 | 2025–2030 | Прогнозные/перспективные данные |

## Датасет 2016–2020

**Путь:**
- `datasets/university_all_years_2016_2020.csv`
- `dataset_2016_2020/university_all_years_2016_2020.csv` (копия)

**Язык колонок:** Русский (кириллица)

**Заголовки:**

| Колонка | Описание | Пример |
|---------|---------|--------|
| `ID студента` | Уникальный код студента в формате `STUxxxxxx` | `STU000736` |
| `ФИО` | Фамилия, имя, отчество студента | `Козлов Никита Иванович` |
| `Год поступления` | Дата поступления в формате `дд.мм.гггг` | `21.01.2016` |
| `Статус` | Текущий статус студента | `выпущен`, `обучается`, `отчислен` |
| `Средний балл` | Средний балл успеваемости | `3.71` или `"3,71"` (запятая как разделитель!) |
| `Год выпуска` | Год окончания обучения | `2022` |
| `Город` | Город проживания | `Томск`, `СПб`, `Новосибирск` |
| `Комментарий` | Свободный текст (может быть пустым) | `повтор`, `acadем отпуск` |

**Особенности:**
- Средний балл иногда записан с запятой вместо точки (`"3,71"`) — система должна уметь это обрабатывать.
- Дата поступления в российском формате `дд.мм.гггг`.
- Статусы в нижнем регистре: `"выпущен"`, `"обучается"`, `"отчислен"`.

## Датасет 2020–2024

**Путь:**
- `datasets/university_all_years_2020_2024.csv`
- `dataset_2020_2024/university_all_years_2020_2024.csv` (копия)

**Язык колонок:** Английский (латиница)

**Заголовки:**

| Колонка | Описание | Соответствует колонке 2016–2020 | Пример |
|---------|---------|-------------------------------|--------|
| `student_id` | Уникальный код студента | `ID студента` | `STU000663` |
| `full name` | ФИО студента | `ФИО` | `Кузнецов Егор Сергеевна` |
| `enrollment_date` | Дата поступления (смешанный формат!) | `Год поступления` | `2021-12-07` или `05.10.2022` |
| `student_status` | Текущий статус | `Статус` | `выпущен`, `обучается` |
| `avg_score` | Средний балл (может быть пустым) | `Средний балл` | `3.45` или пусто |
| `graduation_year` | Год выпуска | `Год выпуска` | `2026` |
| `faculty` | Факультет | — | `Математика` |
| `source_system` | Источник данных (откуда выгружено) | — | `LMS`, `DeanOffice` |

**Особенности:**
- Дата поступления записана в **трёх разных форматах** в одном файле: `2021-12-07` (ISO), `05.10.2022` (российский), пробелы в именах.
- Поле `avg_score` может быть пустым (не у всех студентов есть оценки).
- Появляется новая колонка `faculty` (факультет) — её нет в датасете 2016–2020.

## Датасет 2025–2030

**Путь:**
- `datasets/university_all_years_2025_2030.csv`
- `dataset_2025_2030/university_all_years_2025_2030.csv` (копия)

**Язык колонок:** Смешанный (английская аббревиатура или транслитерация)

**Заголовки:**

| Колонка | Описание | Соответствует | Пример |
|---------|---------|---------------|--------|
| `UID` | Уникальный идентификатор | `student_id`, `ID студента` | `STU000248` |
| `StudentName` | ФИО | `full name`, `ФИО` | `Иванов Татьяна Алексеевич` |
| `Admission` | Дата поступления | `enrollment_date`, `Год поступления` | `02.08.2027`, `03/20/2030` |
| `Current Status` | Статус (с пробелом в названии!) | `student_status`, `Статус` | `Учится`, `graduated` |
| `Score` | Балл | `avg_score`, `Средний балл` | `3.85` или пусто |
| `Graduation` | Год выпуска | `graduation_year`, `Год выпуска` | `2033` |
| `Department` | Кафедра/факультет | `faculty` | `История` |
| `is_budget` | Бюджетник ли | — | `yes`, `no` |
| `notes` | Заметки | — | `manual_fix`, `reimport` |

**Особенности:**
- Статусы смешаны: и английские (`graduated`), и русские (`Учится`).
- Дата поступления в американском формате `03/20/2030` (месяц/день/год).
- Колонка называется `Current Status` (с пробелом внутри) — это проверяет работу с нестандартными именами.
- Самый большой датасет (~9 674 строк), покрывает прогнозный период 2025–2030.

## Папки dataset_* против папки datasets/

В проекте существуют две структуры хранения одних и тех же файлов:

| Расположение | Назначение |
|-------------|-----------|
| `datasets/*.csv` | Основная папка, используется для загрузки в систему |
| `dataset_2016_2020/*.csv` | Отдельные папки, созданные скриптом `generate_uni_data.py` |
| `dataset_2020_2024/*.csv` | (то же самое) |
| `dataset_2025_2030/*.csv` | (то же самое) |

Содержимое файлов идентично — это просто разные места хранения. Для загрузки в систему используйте файлы из папки `datasets/`.

## Генератор датасетов (scripts/generate_uni_data.py)

Интерактивный Python-скрипт для генерации новых тестовых датасетов любого размера и периода.

**Требования:**
```bash
pip install faker rich
```

**Запуск:**
```bash
python scripts/generate_uni_data.py
```

**Что спрашивает:**
1. Начальный год симуляции (например, 2020)
2. Конечный год (например, 2024)
3. Количество поступающих в каждый год (например, 150)
4. Количество преподавателей (например, 45)
5. Формат: один файл за все годы или отдельный файл на каждый год

**Что генерирует:** CSV-файл с колонками:

| Колонка | Описание |
|---------|---------|
| `Год отчета` | За какой год запись |
| `ID Студента` | UUID студента (уникальный) |
| `ФИО студента` | Случайное имя (Faker, ru_RU) |
| `Факультет` | Один из 4 факультетов |
| `Курс` | Текущий курс (1–5) |
| `Статус` | `Обучается`, `Отчислен`, `В академе`, `Выпускник` |
| `Год зачисления` | Год поступления в вуз |
| `Предмет` | Дисциплина (зависит от факультета) |
| `Оценка` | Балл: 2, 3, 4, 5 |
| `ФИО преподавателя` | Случайное имя из штата |

**Алгоритм симуляции:**
- Студенты сохраняют свой ID между годами (нет дублирования записей).
- Каждый год: поступают новые первокурсники, старые переходят на следующий курс.
- Вероятности изменения статуса: 90% — переход, 5% — отчисление, 5% — академический отпуск.
- Студенты в академотпуске возвращаются с вероятностью 50%.
- После 5-го курса — статус «Выпускник», запись архивируется.

**Факультеты и предметы:**
- ИТ: Алгоритмы, БД, Go, ML, Архитектура ЭВМ
- Экономика: Микро/Макроэкономика, Бухучёт, Налоги, Финансы
- Юриспруденция: Уголовное, Гражданское право, ТГП, Криминалистика
- Медицина: Анатомия, Фармакология, Хирургия, Латынь, Биохимия

## Назначение датасетов

| Датасет | Роль в системе |
|---------|---------------|
| `2016_2020` | **Обучение / тестирование**: исторические данные для проверки корректности маппинга колонок |
| `2020_2024` | **Валидация**: современные данные с иными именами колонок |
| `2025_2030` | **Прогноз / демо**: перспективные данные для демонстрации работы с будущими периодами |

Система не разделяет датасеты по назначению — все они загружаются и обрабатываются одинаково. «Назначение» — это контекст для понимания дипломного проекта.

## Ключевая особенность: разные схемы колонок

Одна из главных задач системы — автоматическое определение смысла колонок при помощи LLM.

**Пример задачи:**

| Датасет 2016 | Датасет 2020 | Датасет 2025 | Что имеется в виду |
|-------------|-------------|-------------|-------------------|
| `ID студента` | `student_id` | `UID` | Уникальный номер студента |
| `ФИО` | `full name` | `StudentName` | Полное имя |
| `Год поступления` | `enrollment_date` | `Admission` | Дата/год поступления |
| `Статус` | `student_status` | `Current Status` | Текущий статус |
| `Средний балл` | `avg_score` | `Score` | Средний балл |

LLM получает первые 5 строк данных и список уникальных значений и определяет соответствие. Результат кэшируется в поле `Dataset.ColumnMapping` — повторный вызов LLM не нужен.

## Как система обрабатывает данные

```
Загрузка CSV/Excel
        │
        ▼
Парсинг → массив строк JSON
{
  "ID студента": "STU000736",
  "ФИО": "Козлов Никита Иванович",
  "Год поступления": "21.01.2016",
  "Статус": "выпущен",
  ...
}
        │
        ▼
LLM маппинг (один раз, потом кэш)
{
  "student_id": "ID студента",
  "full_name": "ФИО",
  "enrollment_year": "Год поступления",
  "status": "Статус",
  "score": "Средний балл",
  "active_statuses": ["обучается", "учится", ...]
}
        │
        ▼
Детерминированный подсчёт (без LLM):
- Дедупликация по (studentID + год)
- Подсчёт total, active, avg_score по периодам
        │
        ▼
Мердж в GlobalMetric (с захватом периодов)
        │
        ▼
LLM-инсайты (один раз на мердж):
- summary: "2 400 студентов, рост на 12%..."
- trends: ["Рост в 2022", ...]
- anomalies: ["Всплеск отчислений в 2021", ...]
```
