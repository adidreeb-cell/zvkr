## Обзор архитектуры

Frontend представляет собой **одностраничное приложение** (SPA — Single Page Application), построенное на Preact. Это означает, что браузер загружает один HTML-файл, а затем все переходы между «страницами» происходят без перезагрузки — JavaScript меняет содержимое прямо на месте.

Пользователь не замечает этого различия: для него это обычный сайт с быстрой навигацией.

## Маршрутизация

Маршрутизация настраивается в `App.tsx` через `preact-router`.

| Путь | Компонент | Доступ | Описание |
|------|-----------|--------|---------|
| `/login` | `Login` | Публичный | Форма входа |
| `/setup` | `Setup` | Публичный | Первоначальная настройка (только при первом запуске) |
| `/` | `Dashboard` | Авторизованные | Главный дашборд с KPI и списком датасетов |
| `/upload` | `Upload` | analyst, admin | Загрузка файлов |
| `/dataset/:id` | `DatasetView` | Авторизованные | Просмотр данных и чат с ИИ |
| `/admin/users` | `AdminUsers` | Только admin | Управление пользователями |
| `/analytics/advanced` | `AdvancedAnalytics` | Авторизованные | Расширенная аналитика и графики |
| `/settings` | `Settings` | Только admin | Настройки системы, LLM, cron |

Защищённые маршруты проверяют наличие JWT-токена в `localStorage`. При отсутствии — редирект на `/login`.

## API-клиент

**Файл:** `src/api/client.ts`

Тонкая обёртка над стандартным `fetch`, которая:
1. Добавляет базовый URL `/api/v1`.
2. Автоматически вставляет `Authorization: Bearer <token>` из `localStorage`.
3. Перехватывает ответы `401 Unauthorized` и делает редирект на `/login`.

```typescript
// Пример использования
const res = await apiFetch("/datasets");
const datasets = await res.json();

// POST-запрос
const res = await apiFetch("/datasets/1/chat", {
  method: "POST",
  body: JSON.stringify({ message: "Сколько студентов?", use_code: true })
});
```

## Страницы

### Dashboard (`/`) — `pages/Dashboard.tsx`

**Что отображает:** Главная страница приложения.

**Получает данные с бэкенда:**
- `GET /api/v1/datasets` — список загруженных файлов
- `GET /api/v1/analytics/metrics` — KPI-метрики последнего периода
- `GET /api/v1/news` — ленту образовательных новостей
- `GET /api/v1/analytics/status` — статус обработки (сколько файлов в очереди)

**Элементы интерфейса:**
- **KPI-карточки** (3 штуки): «Контингент», «Активные студенты», «Средний балл»
- **Год статистики** — бейдж рядом с заголовком показывает, за какой год данные
- **Последнее обновление** — время последней синхронизации
- **Кнопка обновления** — иконка стрелок, обновляет данные без перезагрузки страницы
- **Кнопка AI-Аналитика** — переход на страницу графиков
- **Кнопка «Синхронизация»** — запускает ИИ-обработку файлов (только для admin/analyst). Красный бейдж показывает количество файлов в очереди. Деактивируется, когда очередь пуста
- **Список датасетов** — карточки с именем файла, источником, датой и индикатором обработки (зелёная точка)
- **Лента новостей** — последние 4 новости от Минпросвещения с заголовком и источником

**Автообновление:** Каждые 15 секунд в фоне запрашиваются свежие данные (тихое обновление без мигания).

### Dataset View (`/dataset/:id`) — `pages/DatasetView.tsx`

**Что отображает:** Детальный просмотр конкретного датасета.

**Получает данные с бэкенда:**
- `GET /api/v1/datasets/:id` — полные данные датасета
- `GET /api/v1/datasets/:id/chat` — история сообщений

**Вкладки:**
- **«Аналитика»** — компонент `ChatInterface` для общения с ИИ
- **«Данные»** — компонент `DataGrid` для табличного просмотра всех строк

**Кнопки скачивания:**
- Excel: `GET /api/v1/export/excel/:id`
- PDF: `GET /api/v1/export/pdf/:id`

### Chat Interface — `components/chat/ChatInterface.tsx`

**Что отображает:** Интерфейс чата с ИИ внутри страницы датасета.

**Получает данные:** Вся переписка хранится на сервере — при открытии загружается история.

**Пользовательские действия:**

| Элемент | Описание |
|---------|---------|
| Поле ввода | Текстовый вопрос на русском языке |
| Тумблер «Режим кода» | Включает Python Code Interpreter |
| Тумблер «Контекст новостей» | Добавляет новости в промпт (только в текстовом режиме) |
| Кнопка «Отправить» | Отправка `POST /api/v1/datasets/:id/chat` |

**Отображение ответа:**
- Текстовые ответы рендерятся как Markdown (жирный, списки, таблицы).
- В режиме кода показывается:
  - Сгенерированный Python-код с подсветкой синтаксиса
  - JSON-результат выполнения
  - Текстовый отчёт от LLM
  - Графики из поля `charts` в JSON (через `ChartRenderer`)

### Upload (`/upload`) — `pages/Upload.tsx`

**Что отображает:** Страница загрузки файлов.

**Пользовательские действия:**
- Перетаскивание файла (drag-and-drop) или выбор через диалог
- Поддерживаемые форматы: `.xlsx`, `.xls`, `.csv`
- После загрузки (`POST /api/v1/upload`) показывает результат: количество строк, имя файла

### Advanced Analytics (`/analytics/advanced`) — `pages/AdvancedAnalytics.tsx`

**Что отображает:** Расширенная аналитическая панель.

**Получает данные:**
- `GET /api/v1/analytics/advanced?from=YYYY&to=YYYY` — данные временного ряда с фильтром

**Элементы:**
- **Фильтр по датам** — поля «С» и «По» для выбора диапазона лет
- **Линейный график** (Recharts) — динамика числа студентов по годам
- **Разбивка по статусам** — столбчатая диаграмма (обучается / отчислен / в академе)
- **Тренды** — текстовые выводы LLM (список строк)
- **Аномалии** — отклонения, выявленные LLM

### Admin Users (`/admin/users`) — `pages/AdminUsers.tsx`

**Что отображает:** Управление учётными записями.

**Получает данные:**
- `GET /api/v1/users` — список всех пользователей

**Пользовательские действия:**
- Создание пользователя (`POST /api/v1/users/add`): имя + роль → сервер возвращает автосгенерированный пароль
- Удаление пользователя (`POST /api/v1/users/remove`): нельзя удалить себя
- Список показывает: ID, логин, роль, дату создания

### Settings (`/settings`) — `pages/Settings.tsx`

**Что отображает:** Системные настройки (только для администратора).

**Получает данные:**
- `GET /api/v1/settings` — текущие настройки (секреты скрыты масками)

**Пользовательские действия:**
- Выбор LLM-провайдера (OpenRouter / Ollama / YandexGPT / GigaChat)
- Ввод API-ключей для выбранного провайдера
- Настройка cron-выражений для каждой фоновой задачи
- Включение/выключение модулей (IMAP, Moodle, ERP, Эмулятор)
- Настройка RSS-источников новостей
- Кнопка «Сохранить» → `POST /api/v1/settings` → сервер перезапускается через 5 секунд

### Login (`/login`) — `pages/Login.tsx`

**Что отображает:** Форма входа в систему.

**Пользовательские действия:**
- Ввод логина и пароля
- При успехе (`POST /api/v1/login`): JWT-токен и роль сохраняются в `localStorage`, редирект на главную

---

### Setup (`/setup`) — `pages/Setup.tsx`

**Что отображает:** Мастер первого запуска.

**Логика:**
1. Запрашивает `GET /api/v1/system/setup` — получает автосгенерированные логин и пароль.
2. Показывает их пользователю (единственный раз!).
3. После подтверждения вызывает `POST /api/v1/system/setup/complete` — пароль стирается из памяти сервера навсегда.

## Компоненты

### Layout (`components/ui/Layout.tsx`)

Общий каркас всех защищённых страниц:
- **Боковая навигация** (sidebar): ссылки на дашборд, загрузку, аналитику, настройки
- **Шапка** (header): имя пользователя, его роль, кнопка выхода
- Адаптивный дизайн: на мобильных боковая панель скрывается

### DataGrid (`components/data/DataGrid.tsx`)

Таблица для отображения больших датасетов:
- **Виртуальный скролл** — в DOM находятся только видимые строки, остальные не рендерятся (для производительности)
- **Сортировка** — клик по заголовку колонки меняет порядок
- **Пагинация** — постраничная навигация

### ChartRenderer (`components/ui/ChartRenderer.tsx`)

Универсальный рендерер графиков. Принимает JSON-объект из поля `charts` Python-результата и отображает нужный тип:
- `type: "line"` → линейный график (Recharts `LineChart`)
- `type: "bar"` → столбчатая диаграмма (Recharts `BarChart`)
- `type: "pie"` → круговая диаграмма

### Icons (`components/ui/Icons.tsx`)

Коллекция SVG-иконок, используемых в интерфейсе. Содержит анимированный `Spinner` для состояний загрузки.

### Skeleton (`components/ui/Skeleton.tsx`)

Компоненты-заглушки, отображаемые пока данные загружаются. Имитируют форму будущего контента с анимацией мерцания.

## Типы данных

**Файл:** `src/types/index.ts`

```typescript
interface Dataset {
  ID?: string;
  id?: string;
  source: string;       // "upload" | "email" | "moodle" | "erp_database"
  name: string;
  created_at: string;
  is_processed?: boolean;
  headers?: string[] | string;
  data?: any[] | string;

  total_students: number;   // Всего студентов
  active_students: number;  // Активно обучающихся
  average_score: number;    // Средний балл

interface ChatMessage {
  role: Role;              // "user" | "bot" | "system"
  content: string;         // Текст (Markdown)

type Role = "system" | "user" | "bot";

interface User {
  id: number;
  username: string;
  role: string;       // "admin" | "analyst" | "user"
  created_at: string;
}
```

## Процесс аутентификации

1. **Первый запуск**: Сервер создаёт admin-аккаунт и перенаправляет браузер на `/setup`. Пользователь видит временный пароль.
2. **Вход**: `POST /login` → сервер возвращает `{token, role}` → оба сохраняются в `localStorage`.
3. **Защищённые маршруты**: При каждом запросе `apiFetch` автоматически добавляет `Authorization: Bearer <token>`.
4. **Ошибка 401**: `apiFetch` перехватывает ответ → удаляет токен → редирект на `/login`.
5. **Срок жизни токена**: 72 часа (3 дня). После истечения — автоматический выход.

## Управление состоянием

Preact использует хуки для управления состоянием — это встроенный механизм, не требующий отдельных библиотек:

```typescript
// Состояние страницы
const [datasets, setDatasets] = useState<Dataset[]>([]);
const [loading, setLoading] = useState(true);

// Загрузка данных при монтировании компонента
useEffect(() => {
  apiFetch("/datasets").then(r => r.json()).then(setDatasets);
}, []);
```

Общего хранилища состояния (Redux, Zustand) нет — каждая страница управляет своими данными. Роль пользователя и токен хранятся в `localStorage` (браузерное постоянное хранилище).

## Стилизация

**TailwindCSS v4** с настроенной дизайн-системой:

- **Тёмная тема** по умолчанию (светлой нет)
- **Цветовая палитра**: серые оттенки + акценты (синий, зелёный, фиолетовый)
- **Эффекты стекла**: `backdrop-blur` + полупрозрачные фоны
- **Плавные переходы**: `transition-all duration-200` на интерактивных элементах
- **Скругления**: `rounded-2xl` (16px) — единый стиль карточек и кнопок

Для кастомных компонентов используются CSS-переменные (`--color-surface`, `--color-border` и т.д.), определённые в `index.css`.
