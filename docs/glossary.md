## Технические термины

### API (Application Programming Interface)
Интерфейс программирования приложений — набор правил, по которым разные программы общаются друг с другом. В ZVKR: браузер общается с сервером через REST API, отправляя запросы и получая JSON-ответы.

**Где встречается:** `internal/routes/routes.go`, `docs/api.md`

---

### bcrypt
Алгоритм необратимого шифрования паролей. После того как пароль захэшируется через bcrypt, получить исходный пароль обратно невозможно — система сравнивает хэши при входе. В ZVKR используется с параметром «стоимость» (cost) = 14, что делает подбор пароля брутфорсом непрактичным.

**Где встречается:** `internal/services/auth/jwt.go` (функции `HashPassword`, `CheckPassword`)

---

### BoltDB
Встроенная база данных «ключ-значение» для Go. В отличие от SQL-баз не имеет таблиц — данные хранятся как пары «ключ → бинарные данные». В ZVKR используется для кэширования новостей: быстро читается, не требует сложных запросов.

**Где встречается:** `cmd/gate/main.go`, `internal/integrations/news.go`

---

### CORS (Cross-Origin Resource Sharing)
Механизм безопасности браузера, который ограничивает запросы к другим доменам. В режиме разработки фронтенд (порт 5173) запрашивает данные у бэкенда (порт 6331) — браузер по умолчанию это блокирует. CORS-заголовки разрешают такие запросы. В ZVKR настроен как `AllowOrigins: "*"` (все домены разрешены).

**Где встречается:** `cmd/gate/main.go`

---

### CSV (Comma-Separated Values)
Формат текстового файла, где данные разделены запятыми. Первая строка — заголовки колонок, остальные строки — данные. Широко используется для выгрузки из 1С, Excel, любых баз данных.

**Пример:**
```
ФИО,Статус,Балл
Иванов И.И.,обучается,4.5
Петров П.П.,отчислен,
```

**Где встречается:** `internal/services/excel/parser.go`, папка `datasets/`

---

### Embed (встраивание ресурсов)
Механизм языка Go, позволяющий включить содержимое файлов прямо в исполняемый бинарник во время компиляции. В ZVKR так встроены: фронтенд (`internal/routes/dist/`), Python WASM (`sandbox/python.wasm`), шрифт Arial (`export/fonts/arial.ttf`).

**Где встречается:** `internal/routes/routes.go` (`//go:embed dist/*`), `internal/services/sandbox/wazero.go`

---

### Fiber
Веб-фреймворк для языка Go, похожий на Express.js (Node.js). Позволяет удобно объявлять HTTP-маршруты, подключать middleware, отдавать JSON и статические файлы.

**Где встречается:** `cmd/gate/main.go`, `internal/routes/routes.go`

---

### Go (Golang)
Язык программирования от Google. Особенности: быстрый, компилируется в один исполняемый файл без зависимостей, отличная поддержка многопоточности (горутины). В ZVKR используется для всего бэкенда.

**Где встречается:** все файлы `.go`

---

### Горутина (goroutine)
Лёгкий поток выполнения в Go. Несколько горутин могут работать одновременно. В ZVKR используются для параллельного анализа чанков датасета и параллельного получения новостей.

**Где встречается:** `internal/handlers/dataset.go` (Map-Reduce), `internal/services/analytics/metrics.go`

---

### GORM
ORM-библиотека для Go (Object-Relational Mapping). Позволяет работать с базой данных через Go-структуры, не писать SQL вручную. `db.Find(&users)` — это GORM автоматически выполняет `SELECT * FROM users`.

**Где встречается:** все файлы в `internal/models/`, `internal/database/connect.go`

---

### IMAP (Internet Message Access Protocol)
Протокол для чтения электронной почты с сервера. В ZVKR используется для автоматического скачивания Excel-вложений из писем на почтовый ящик деканата.

**Где встречается:** `internal/services/email/imap.go`

---

### JSON (JavaScript Object Notation)
Текстовый формат обмена данными. Использует фигурные скобки `{}` для объектов и квадратные `[]` для массивов. В ZVKR все данные между бэкендом и фронтендом передаются в JSON.

**Пример:**
```json
{
  "total_students": 2850,
  "active_students": 1920,
  "average_score": 4.12
}
```

---

### JWT (JSON Web Token)
Токен авторизации — зашифрованная строка, которая содержит информацию о пользователе (ID, роль) и имеет срок жизни. После входа клиент получает JWT и прикладывает его к каждому запросу. Сервер проверяет подпись без обращения к БД.

**Где встречается:** `internal/services/auth/jwt.go`, `internal/middleware/auth_middleware.go`

---

### LLM (Large Language Model)
Большая языковая модель — тип искусственного интеллекта, обученный на огромных объёмах текста. Умеет понимать вопросы на естественном языке и генерировать связные ответы. В ZVKR используется для анализа данных, маппинга колонок и генерации Python-кода.

**Примеры моделей:** GPT-4, Gemma, LLaMA, GigaChat, YandexGPT

**Где встречается:** `internal/services/llm/`, `internal/services/ai_pipeline.go`

---

### Map-Reduce
Алгоритмическая парадигма обработки больших данных. «Map» — разбить задачу на части и обработать параллельно. «Reduce» — объединить результаты. В ZVKR применяется к анализу больших датасетов: LLM анализирует по 50 строк, затем объединяет выводы.

**Где встречается:** `internal/handlers/dataset.go`, `internal/services/ai_pipeline.go`

---

### Middleware (промежуточное ПО)
Функции, которые вызываются до/после основного обработчика запроса. В ZVKR middleware проверяет JWT-токен (аутентификация) и роль пользователя (авторизация) перед вызовом бизнес-логики.

**Где встречается:** `internal/middleware/auth_middleware.go`

---

### Moodle
Система управления обучением (LMS — Learning Management System). Используется во многих университетах для создания онлайн-курсов, тестов и учёта успеваемости. ZVKR может получать данные из Moodle через API.

**Где встречается:** `internal/services/moodle/moodle.go`

---

### OpenRouter
Платформа, предоставляющая доступ к множеству языковых моделей через единый API (совместимый с OpenAI). Позволяет выбирать модель по критерию «цена/качество». В ZVKR используется как провайдер LLM по умолчанию.

**Где встречается:** `internal/services/llm/openrouter.go`

---

### ORM (Object-Relational Mapping)
Технология, которая «переводит» объекты программного кода в строки базы данных и обратно. Вместо SQL-запроса `INSERT INTO users ...` программист пишет `db.Create(&user)`.

**Где встречается:** весь проект через GORM

---

### Polling (опрос)
Техника обновления данных: клиент периодически спрашивает сервер «есть ли что-то новое?» вместо того, чтобы сервер сам уведомлял клиента. В ZVKR дашборд опрашивает сервер каждые 15 секунд.

**Где встречается:** `frontend/src/pages/Dashboard.tsx`

---

### Preact
Компактная альтернатива React для создания интерактивных веб-интерфейсов. Использует компонентный подход: UI разбивается на независимые части (компоненты), которые перерисовываются при изменении данных.

**Где встречается:** весь `frontend/src/`

---

### REST API (Representational State Transfer)
Архитектурный стиль веб-API. Каждый ресурс (датасет, пользователь) имеет свой URL. Операции: GET (получить), POST (создать/отправить), PUT (обновить), DELETE (удалить).

**Где встречается:** `internal/routes/routes.go`

---

### RSS (Really Simple Syndication)
Формат XML-файла для публикации новостей. Сайты публикуют RSS-ленту, а программы подписываются на неё и получают обновления. В ZVKR парсятся RSS-ленты Минпросвещения и Рособрнадзора.

**Где встречается:** `internal/integrations/news.go`

---

### SHA-1
Алгоритм хэш-функции, производящий 160-битное (40 hex-символов) «отпечаток» строки. Одинаковые входные данные дают одинаковый хэш. В ZVKR используется для идентификации строк датасета без явного ID студента.

**Где встречается:** `internal/services/analytics/metrics.go` (функция `makeStableRowSignature`)

---

### SQLite
Компактная реляционная база данных, хранящаяся в одном файле. Не требует отдельного сервера — встраивается прямо в приложение. В ZVKR используется как основная база данных (`doctor.db`).

**Где встречается:** `internal/database/connect.go`

---

### SPA (Single Page Application)
Одностраничное приложение — веб-сайт, где браузер загружает одну HTML-страницу, а переходы между «страницами» происходят без перезагрузки через JavaScript. В ZVKR фронтенд реализован как SPA на Preact.

**Где встречается:** `frontend/src/App.tsx`, `internal/routes/routes.go` (SPA fallback)

---

### TailwindCSS
CSS-фреймворк, в котором стили задаются через классы прямо в HTML/JSX. Вместо написания отдельных CSS-файлов используются утилитарные классы: `text-white`, `p-4`, `rounded-xl`.

**Где встречается:** весь `frontend/src/`

---

### TypeScript
Расширение языка JavaScript с системой типов. Позволяет указывать, какого типа данные ожидаются (число, строка, объект) — это помогает находить ошибки до запуска кода.

**Где встречается:** весь `frontend/src/` (файлы `.ts`, `.tsx`)

---

### Wazero
Go-библиотека для запуска WebAssembly-кода без внешних зависимостей. В ZVKR используется для запуска Python-интерпретатора (скомпилированного в WebAssembly) внутри Go-процесса.

**Где встречается:** `internal/services/sandbox/wazero.go`

---

### WebAssembly (WASM)
Формат байт-кода, который может выполняться в браузере или специальном рантайме (Wazero). Обеспечивает почти нативную скорость работы и изоляцию. В ZVKR Python скомпилирован в WASM для безопасного выполнения.

**Где встречается:** `internal/services/sandbox/python.wasm`

---

### XLSX
Формат файлов Microsoft Excel (начиная с Office 2007). Технически — ZIP-архив с XML-файлами внутри. В ZVKR файлы XLSX парсятся библиотекой Excelize.

**Где встречается:** `internal/services/excel/parser.go`, `internal/services/export/excel.go`

---

## Термины предметной области

### Контингент
Общее число студентов, обучающихся в учебном заведении в данный момент. В ZVKR: `total_students` — это весь контингент (включая отчисленных и выпускников за период), `active_students` — только активно обучающиеся.

**Где встречается:** `internal/models/metrics.go`, дашборд

---

### Академический отпуск (Академотпуск)
Временное прекращение обучения по уважительным причинам (болезнь, семейные обстоятельства). Студент сохраняет статус и возвращается после окончания отпуска. В датасетах встречается как `"В академе"`, `"academ отпуск"`.

**Где встречается:** `datasets/`, `internal/services/emulator/emulator.go`

---

### Средний балл (GPA)
Среднее арифметическое всех оценок студента. В российской системе: 5 — отлично, 4 — хорошо, 3 — удовлетворительно, 2 — неудовлетворительно. В датасетах хранится как десятичное число: `4.12`.

**Где встречается:** `datasets/`, `internal/services/analytics/metrics.go`

---

### Маппинг колонок
Процесс установления соответствия между названиями колонок в загруженном файле и смысловыми полями системы. Например: колонка `"StudentName"` в файле соответствует полю `full_name` в системе. В ZVKR маппинг выполняется автоматически через LLM.

**Где встречается:** `internal/services/analytics/metrics.go` (функции `detectColumnMapping`, `resolveColumnMapping`)

---

### Период
В ZVKR — год, за который собраны данные. Извлекается из колонки «год поступления». Данные группируются по периодам для построения временного ряда.

**Где встречается:** `internal/services/analytics/metrics.go`

---

### Временной ряд (Time Series)
Последовательность значений, измеренных через равные промежутки времени. В ZVKR: количество студентов, средний балл и распределение по статусам по годам. Используется для построения линейных графиков на странице «Аналитика».

**Где встречается:** `internal/models/metrics.go` (`TimeSeries`), `frontend/src/pages/AdvancedAnalytics.tsx`

---

### ERP (Enterprise Resource Planning)
Система планирования ресурсов предприятия — корпоративное программное обеспечение для управления всеми бизнес-процессами. В образовании: системы учёта студентов, финансов, кадров (1С, SAP). ZVKR может подключаться к базе данных ERP напрямую.

**Где встречается:** `internal/services/erp/erp.go`

---

### Дедупликация
Удаление повторяющихся записей. В датасетах один студент может встречаться несколько раз (разные дисциплины, разные выгрузки). Система считает каждого студента в каждом периоде только один раз.

**Где встречается:** `internal/services/analytics/metrics.go` (функция `deduplicateRows`)

---

### Аномалия
Отклонение от нормального поведения данных. В ZVKR: резкий рост числа отчислений, неожиданное падение среднего балла. Выявляются языковой моделью на основе посчитанных метрик.

**Где встречается:** `internal/models/metrics.go` (`Anomalies`), страница «Аналитика»

---

### Тренд
Устойчивая тенденция изменения показателя во времени. Например: «Число студентов стабильно растёт на 5% в год». В ZVKR тренды формулирует языковая модель.

**Где встречается:** `internal/models/metrics.go` (`Trends`), страница «Аналитика»

---

### Ownership периода
Механизм в ZVKR, при котором каждый год (период) «принадлежит» одному датасету. Если два датасета содержат данные за один год — данные берутся только из первого загруженного. Это предотвращает двойной счёт студентов.

**Где встречается:** `internal/models/metrics.go` (`ProcessedPeriod`), `internal/services/analytics/metrics.go`
