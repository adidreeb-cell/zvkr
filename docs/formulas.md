## Среднее арифметическое (средний балл)

### Описание

Система вычисляет средний балл студентов — самую распространённую статистическую формулу.

### Формула

```math
\bar{x} = \frac{\sum_{i=1}^{n} x_i}{n}
```

Где:
- $\bar{x}$ — средний балл
- $x_i$ — балл $i$-го студента
- $n$ — количество студентов, у которых есть балл (пустые значения исключаются)

### Реализация в коде

**Файл:** `internal/services/analytics/metrics.go`, функция `computeMetricsDeterministic` (строки 278–348)

```go
// Накопление
acc.scoreSum += score   // Сумма баллов
acc.scoreCount++        // Счётчик студентов с баллом

// Вычисление среднего (в методе averageScore())
func (a *periodAccumulator) averageScore() float64 {
    if a.scoreCount == 0 {
        return 0
    }
    return math.Round((a.scoreSum/float64(a.scoreCount))*100) / 100
}
```

**Важная деталь:** Результат округляется до 2 знаков после запятой формулой:

```math
\bar{x}_{rounded} = \frac{\lfloor \bar{x} \times 100 + 0.5 \rfloor}{100}
```

### Пример с числами

Датасет с 5 студентами: `[4.5, 3.0, 5.0, 4.0, 4.5]`

```
sum = 4.5 + 3.0 + 5.0 + 4.0 + 4.5 = 21.0
count = 5
average = 21.0 / 5 = 4.2
rounded = round(4.2 * 100) / 100 = 4.2
```

## Взвешенное среднее при мерже периодов

### Описание

При объединении нескольких датасетов нельзя просто усреднить средние баллы — это математически некорректно. Нужно использовать **взвешенное среднее** через накопленные суммы.

### Почему нельзя усреднять средние?

Пример: датасет A — 10 студентов со средним 4.0, датасет B — 1 студент со средним 5.0.

- Неверно: $(4.0 + 5.0) / 2 = 4.5$
- Верно: $(10 \times 4.0 + 1 \times 5.0) / 11 = 45 / 11 \approx 4.09$

### Формула

```math
\bar{x}_{merged} = \frac{\sum_{j=1}^{k} S_j}{\sum_{j=1}^{k} N_j}
```

Где:
- $S_j$ — сумма баллов в $j$-м периоде (`score_sum`)
- $N_j$ — количество студентов с баллом в $j$-м периоде (`score_count`)
- $k$ — количество периодов

### Реализация

**Файл:** `internal/services/analytics/metrics.go`, функция `recalcMetricsFromTimeSeries` (строки 664–703)

```go
var totalScoreSum float64
var totalScoreCount int64

for _, ts := range timeSeries {
    // Если у периода есть накопленные данные (новый формат)
    if ts.ScoreCount > 0 {
        totalScoreSum += ts.ScoreSum
        totalScoreCount += ts.ScoreCount
        continue
    }
    // Обратная совместимость: восстанавливаем сумму из среднего
    if ts.TotalStudents > 0 {
        totalScoreSum += ts.AverageScore * float64(ts.TotalStudents)
        totalScoreCount += ts.TotalStudents
    }
}

if totalScoreCount > 0 {
    avgScore = math.Round((totalScoreSum/float64(totalScoreCount))*100) / 100
}
```

### Пример с числами

| Период | Студентов с баллом | Сумма баллов | Среднее |
|--------|-------------------|-------------|---------|
| 2019 | 500 | 2050 | 4.10 |
| 2020 | 600 | 2400 | 4.00 |
| 2021 | 550 | 2255 | 4.10 |

Неверный способ: $(4.10 + 4.00 + 4.10) / 3 = 4.067$

Верный способ:
```
totalScoreSum = 2050 + 2400 + 2255 = 6705
totalScoreCount = 500 + 600 + 550 = 1650
average = 6705 / 1650 ≈ 4.064
```

## Дедупликация строк

### Описание

В реальных выгрузках один студент может встречаться несколько раз (например, несколько оценок за год). Система исключает дубликаты перед подсчётом.

### Алгоритм

Для каждой строки вычисляется составной ключ:

```math
key = studentID + "|" + period
```

Если такой ключ уже встречался — строка пропускается.

### Приоритет формирования ключа студента

```
1. Если есть student_id колонка → "id:" + lowercase(student_id)
2. Если есть full_name колонка → "name:" + lowercase(full_name)
3. Если ничего нет → "rowhash:" + SHA1(всех полей строки)
```

### Реализация

**Файл:** `internal/services/analytics/metrics.go`, функция `deduplicateRows` (строки 366–398)

```go
seen := make(map[string]bool, len(data))

for _, row := range data {
    period := extractPeriod(row, mapping)
    studentKey := buildStudentKey(row, mapping)

    if studentKey == "" {
        studentKey = "rowhash:" + makeStableRowSignature(row)
    }

    compositeKey := studentKey + "|" + period
    if seen[compositeKey] {
        skippedDuplicates++
        continue
    }

    seen[compositeKey] = true
    result = append(result, row)
}
```

### Пример с числами

Входные данные (3 строки, 2 уникальных студента за год):
```
STU001 | Иванов | 2022 | обучается | 4.5
STU001 | Иванов | 2022 | обучается | 3.8  ← дубликат (тот же студент, тот же год)
STU002 | Петров | 2022 | отчислен  | —
```

После дедупликации:
```
STU001 | Иванов | 2022 | обучается | 4.5  ← первая строка (сохранена)
STU002 | Петров | 2022 | отчислен  | —
```

Логика: в одном периоде каждый студент считается один раз.

## Извлечение периода (года)

### Описание

Поле «год поступления» приходит в разных форматах. Система должна извлечь 4-значный год из любого из них.

### Алгоритм (приоритет)

```
1. Если строка начинается с 4 цифр (yyyy...) → берём первые 4 символа
2. Иначе разбиваем по разделителям: '.', '/', '-'
   и ищем 4-значное число среди частей (справа налево)
3. Если год не найден → возвращаем "unknown"
```

### Реализация

**Файл:** `internal/services/analytics/metrics.go`, функция `extractPeriod` (строки 435–466)

### Примеры

| Входное значение | Результат |
|-----------------|---------|
| `"21.01.2016"` | `"2016"` |
| `"2021-12-07"` | `"2021"` |
| `"03/20/2030"` | `"2030"` |
| `"2022"` | `"2022"` |
| `"05.10.2022 "` (с пробелом) | `"2022"` |
| `""` или null | `"unknown"` |

## Хэш строки для идентификации

### Описание

Когда в строке нет ни ID студента, ни имени — система генерирует детерминированный хэш всей строки. Это позволяет всё равно дедуплицировать идентичные строки.

### Алгоритм

1. Сортируем ключи строки по алфавиту.
2. Конкатенируем: `"ключ1=значение1;ключ2=значение2;..."` (всё в нижнем регистре).
3. Вычисляем SHA-1 от получившейся строки.
4. Возвращаем hex-представление.

### Формула (псевдокод)

```math
hash = \text{SHA1}\left(\text{join}_{sorted\_keys}\left(k_i + "=" + \text{lower}(v_i) + ";"\right)\right)
```

### Реализация

**Файл:** `internal/services/analytics/metrics.go`, функция `makeStableRowSignature` (строки 416–433)

```go
sum := sha1.Sum([]byte(b.String()))
return hex.EncodeToString(sum[:])
```

### Почему SHA-1?

SHA-1 здесь используется не для криптографической безопасности, а для получения компактного детерминированного идентификатора строки. Коллизии маловероятны для небольших датасетов.

## Генерация тестовых данных (случайный дрейф)

### Описание

Эмулятор (`EmulatorService`) генерирует «живые» данные с небольшим случайным отклонением от базового значения — имитирует реальные колебания контингента.

### Формула

```math
N_{students} = N_{base} + \Delta
```

```math
\Delta \sim \text{Uniform}(-15, +35)
```

Где:
- $N_{base} = 1500$ — базовое число студентов
- $\Delta$ — случайное целое число в диапазоне $[-15, +35]$

### Реализация

**Файл:** `internal/services/emulator/emulator.go`, функция `GenerateMockDrift` (строки 23–67)

```go
baseStudents := 1500
drift := rand.Intn(50) - 15  // от -15 до +34
currentStudents := baseStudents + drift
```

### Пример с числами

При каждом запуске эмулятора возможны значения:
- Минимум: 1500 - 15 = **1485 студентов**
- Максимум: 1500 + 34 = **1534 студента**
- Типичное значение: около 1507 студентов

## Пайплайн предсказания (полный)

Полное описание процесса от загрузки файла до отображения результата.

### 1. Загрузка данных

**Файл:** `internal/services/excel/parser.go`

Входной файл (.xlsx или .csv) парсится построчно. Первая строка — заголовки, остальные — данные.

```
Входной CSV файл
      │
      ▼
Тип файла?
  ├── .csv → csv.ReadAll() → [][]string
  └── .xlsx → excelize.GetRows() → [][]string
      │
      ▼
rowsToMap(): headers[0] + rows[1:] → []map[string]interface{}
```

**Результат:** список строк, где каждая строка — словарь `{имя_колонки: значение}`.

### 2. Предобработка (маппинг колонок)

**Файл:** `internal/services/analytics/metrics.go`, функция `detectColumnMapping`

LLM получает:
- 5 примерных строк данных (в JSON)
- Список уникальных значений каждой колонки (до 50 штук)

И возвращает JSON-маппинг:
```json
{
  "student_id": "ID студента",
  "full_name": "ФИО",
  "enrollment_year": "Год поступления",
  "status": "Статус",
  "score": "Средний балл",
  "active_statuses": ["обучается", "Учится", "учится"]
}
```

Результат кэшируется — повторный вызов LLM не нужен.

### 3. Детерминированный подсчёт метрик

**Файл:** `internal/services/analytics/metrics.go`, функция `computeMetricsDeterministic`

Без участия ИИ, только математика:

```
Для каждой строки:
  1. Определить период (год)
  2. Определить ключ студента (ID или имя или хэш)
  3. Если (ключ + период) уже видели → пропустить (дедупликация)
  4. Иначе:
     - total_students[period]++
     - Если статус ∈ active_statuses → active_students[period]++
     - Если есть балл → score_sum[period] += балл, score_count[period]++
```

### 4. Мердж в GlobalMetric

**Файл:** `internal/services/analytics/metrics.go`, функция `mergeIntoGlobalMetrics`

В транзакции БД обновляется единственная запись глобальных метрик:

```
Для каждого периода из нового датасета:
  Если период уже занят другим датасетом → пропустить
  Иначе → занять период, обновить данные
Пересчитать aggregate (sum over all periods)
```

### 5. Генерация текстовых инсайтов

**Файл:** `internal/services/analytics/metrics.go`, функция `generateTextInsights`

LLM получает точные цифры (посчитанные программно) и генерирует тексты:

```
Промпт → LLM
"Вот точные метрики: {JSON}. Составь summary, trends, anomalies. Верни JSON."

Ответ LLM:
{
  "summary": "...",
  "trends": ["...", "..."],
  "anomalies": ["..."]
}
```

### 6. Code Interpreter (режим чата)

**Файл:** `internal/handlers/dataset.go`, функция `handleCodeAnalysis`

```

**Файл:** `internal/handlers/dataset.go`, функция `handleTextAnalysisSmart`

      ▼
Финальный ответ
```

## Map-Reduce для больших датасетов

### Описание

Языковые модели имеют ограничение на длину входного текста (context window). Если датасет большой — нельзя отправить его целиком. Используется алгоритм Map-Reduce.

### Алгоритм

**Map (параллельный):** Каждый чанк анализируется независимо.

```math
S_i = \text{LLM}(q, D_i), \quad i = 1, \ldots, k
```

Где:
- $q$ — вопрос пользователя
- $D_i$ — $i$-й фрагмент данных (50 строк)
- $S_i$ — краткий вывод по фрагменту
- $k$ — количество фрагментов (максимум 5)

**Reduce (последовательный):** Объединение всех выводов.

```math
\text{Answer} = \text{LLM}(q, [S_1, S_2, \ldots, S_k])
```

### Реализация

**Файл:** `internal/handlers/dataset.go`, функция `handleTextAnalysisSmart` (строки 428–521)

```go
const chunkSize = 50
maxChunks := 5
numChunks := int(math.Ceil(float64(totalRows) / float64(chunkSize)))
if numChunks > maxChunks {
    numChunks = maxChunks
}

// Map: параллельно (горутины)
var wg sync.WaitGroup
for i := 0; i < numChunks; i++ {
    wg.Add(1)
    go func(chunkIdx int) {
        defer wg.Done()
        // анализ chunkData[start:end]
    }(i)
}
wg.Wait()

// Reduce: последовательно
finalAnswer, _ = LLM.AnalyzeData(ctx, finalPrompt, nil)
```

### Пример с числами

Датасет 200 строк, вопрос: «Сколько студентов отчислилось?»

```
Чанк 1 (строки 0-49):   "В первом фрагменте: 3 отчисления"
Чанк 2 (строки 50-99):  "Во втором фрагменте: 2 отчисления"
Чанк 3 (строки 100-149):"В третьем фрагменте: 4 отчисления"
Чанк 4 (строки 150-199):"В четвёртом фрагменте: 1 отчисление"

Reduce: "Всего 10 отчислений в данном датасете"
```

> **Ограничение:** Анализируются первые 250 строк (5 × 50). Для полного анализа больших файлов рекомендуется режим Code Interpreter.
