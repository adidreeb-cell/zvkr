## Общие сведения

**Базовый URL:** `http://localhost:6331/api/v1`

**Формат данных:** JSON (если не указано иное)

**Заголовки для защищённых запросов:**
```
Authorization: Bearer <JWT-токен>
Content-Type: application/json
```

Все ответы с ошибкой содержат поле `"error"` с описанием:
```json
{ "error": "Описание ошибки" }
```

## Аутентификация

### Вход в систему

```
POST /api/v1/login
```

**Тело запроса:**
```json
{
  "username": "admin",
  "password": "ваш-пароль"
}
```

**Успешный ответ (200):**
```json
{
  "token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...",
  "role": "admin"
}
```

**Ошибки:**
- `401` — Пользователь не найден или неверный пароль

**Пример (curl):**
```bash
curl -X POST http://localhost:6331/api/v1/login \
  -H "Content-Type: application/json" \
  -d '{"username":"admin","password":"secret"}'
```

---

### Регистрация

```
POST /api/v1/register
```

**Тело запроса:**
```json
{
  "username": "newuser",
  "password": "mypassword",
  "role": "user"
}
```

**Успешный ответ (201):**
```json
{ "message": "Пользователь успешно создан" }
```

**Ошибки:**
- `409` — Пользователь с таким именем уже существует

---

### Первый запуск — получение пароля admin

```
GET /api/v1/system/setup
```

Работает только при первом запуске (пока `SetupPassword` не пуст).

**Успешный ответ (200):**
```json
{
  "username": "admin",
  "password": "Abc123XyZ456..." 
}
```

**Ошибки:**
- `403` — Система уже инициализирована

---

### Завершить настройку (стереть пароль)

```
POST /api/v1/system/setup/complete
```

**Успешный ответ (200):**
```json
{ "success": true }
```

## Управление пользователями

> Требуется роль `admin`.

### Получить список пользователей

```
GET /api/v1/users
```

**Ответ (200):**
```json
[
  {
    "id": 1,
    "username": "admin",
    "role": "admin",
    "created_at": "2024-01-01T10:00:00Z",
    "updated_at": "2024-01-01T10:00:00Z"
  }
]
```

---

### Создать пользователя

```
POST /api/v1/users/add
```

**Тело:**
```json
{
  "username": "analyst1",
  "role": "analyst"
}
```

**Ответ (201):**
```json
{
  "username": "analyst1",
  "role": "analyst",
  "password": "AutoGenerated32CharPassword..."
}
```

> Пароль генерируется автоматически. Сообщите его пользователю — в системе он хранится только как bcrypt-хэш.

---

### Удалить пользователя

```
POST /api/v1/users/remove
```

**Примечание:** Нельзя удалить собственный аккаунт.

**Тело:**
```json
{ "id": 2 }
```

**Ответ (200):**
```json
{ "message": "Пользователь успешно удален" }
```

### Обновить настройки

```
POST /api/v1/settings
```

**Тело:** Тот же формат, что и в GET. Секретные поля можно опустить или передать маску — они не будут перезаписаны.

**Ответ (200):**
```json
{
  "success": true,
  "message": "Настройки сохранены и применены. СЕРВЕР БУДЕТ ПЕРЕЗАГРУЖЕН"
}
```

> После сохранения сервер перезапускается через 5 секунд для применения новой конфигурации LLM.

## Датасеты

### Загрузить файл

```
POST /api/v1/upload
Content-Type: multipart/form-data
```

**Поле формы:** `file` — Excel (.xlsx, .xls) или CSV файл.

**Ответ (200):**
```json
{
  "id": 5,
  "rows_count": 6928,
  "filename": "university_2020_2024.csv"
}
```

**Ошибки:**
- `400` — Файл не приложен или не удалось разобрать
- `500` — Ошибка записи в базу данных

**Пример (curl):**
```bash
curl -X POST http://localhost:6331/api/v1/upload \
  -H "Authorization: Bearer <token>" \
  -F "file=@university_all_years_2020_2024.csv"
```

---

### Статус синхронизации

```
GET /api/v1/analytics/status
```

**Ответ (200):**
```json
{
  "success": true,
  "is_syncing": true,
  "pending_count": 2,
  "last_updated": "2024-01-15T14:30:00Z"
}
```

- `is_syncing: true` означает, что есть необработанные датасеты (нужна синхронизация)
- `pending_count` — сколько файлов ожидают ИИ-обработки

---

### Принудительная синхронизация

```
GET /api/v1/analytics/sync
```

> Требуется роль `analyst` или `admin`.

Запускает немедленную обработку всех необработанных датасетов. Может занять от нескольких секунд до нескольких минут в зависимости от объёма данных и скорости LLM.

**Ответ (200):**
```json
{
  "success": true,
  "message": "Данные успешно проанализированы и метрики обновлены"
}
```

**Ошибки:**
- `500` — Ошибка при обработке (детали в поле `details`)

### История чата

```
GET /api/v1/datasets/:id/chat
```

**Ответ (200):**
```json
[
  {
    "id": 1,
    "dataset_id": 3,
    "role": "user",
    "content": "Сколько студентов отчислилось в 2022 году?",
    "code_output": "",
    "source_code": "",
    "is_error": false,
    "created_at": "2024-01-01T10:00:00Z"
  },
  {
    "id": 2,
    "dataset_id": 3,
    "role": "bot",
    "content": "## Анализ данных показал...\n\nВ 2022 году было отчислено **45 студентов**...",
    "code_output": "{\"meta\":{\"scanned_rows\":6999,...},\"summary\":{\"expelled_2022\":45},...}",
    "source_code": "import json\n\ndef analyze_data():\n...",
    "is_error": false,
    "created_at": "2024-01-01T10:01:00Z"
  }
]
```

---

### Отправить вопрос ИИ

```
POST /api/v1/datasets/:id/chat
```

**Тело:**
```json
{
  "message": "Сколько студентов отчислилось в 2022 году?",
  "use_code": true,
  "use_news": false
}
```

**Параметры:**
- `message` — вопрос на любом языке
- `use_code` — `true` для Python Code Interpreter, `false` для текстового анализа
- `use_news` — `true` добавляет новости образования в контекст (только в текстовом режиме)

**Ответ в режиме кода (200):**
```json
{
  "reply": "## Анализ показал...\n\nВ 2022 году было **45 отчислений**...",
  "code_output": "{\"meta\":{\"scanned_rows\":6999,\"used_rows\":6990,\"total_rows_expected\":6999},\"summary\":{\"expelled_2022\":45},\"charts\":[]}",
  "source_code": "import json\n\ndef analyze_data():\n    results = {...}\n    ...\nprint(json.dumps(analyze_data()))",
  "dataset_id": 3,
  "mode": "code_analytics"
}
```

**Ответ в текстовом режиме (200):**
```json
{
  "reply": "На основе данных датасета...",
  "dataset_id": 3,
  "mode": "text_simple"
}
```

или для больших датасетов:
```json
{
  "reply": "Объединённый анализ фрагментов...",
  "dataset_id": 3,
  "mode": "text_map_reduce"
}
```

**Ошибки:**
- `404` — Датасет не найден
- `500` — ИИ не смог сгенерировать корректный код за 3 попытки

## Python-песочница

### Выполнить Python-код

```
POST /api/v1/python/run
```

**Тело:**
```json
{
  "code": "import json\nprint(json.dumps({'result': 2 + 2}))"
}
```

**Ответ при успехе (200):**
```json
{
  "status": "success",
  "result": "{\"result\": 4}"
}
```

**Ответ при ошибке (200):**
```json
{
  "status": "error",
  "error": "Runtime Error: exit status 1\nDetails:\nTraceback..."
}
```

> В этом эндпоинте данные датасета не подставляются — выполняется чистый код.

## Новости

### Получить кэшированные новости

```
GET /api/v1/news
```

Публичный эндпоинт. Возвращает последние новости из кэша BoltDB.

**Ответ (200):**
```json
[
  {
    "title": "Минпросвещения утвердило новые стандарты...",
    "link": "https://edu.gov.ru/press/...",
    "published_at": "2024-01-15T09:00:00Z",
    "description": "Министерство просвещения Российской Федерации...",
    "source": "Минпросвещения РФ"
  }
]
```

**Ответ 503** — если новости ещё не загружены (первый запуск):
```json
{ "error": "Новости еще загружаются" }
```

## Коды ошибок

| HTTP-код | Значение | Типичные причины |
|---------|---------|----------------|
| `200` | Успешно | — |
| `201` | Создано | Пользователь или ресурс создан |
| `400` | Неверный запрос | Отсутствует файл, неверный JSON |
| `401` | Не авторизован | Нет токена, просрочен, неверный пароль |
| `403` | Доступ запрещён | Недостаточно прав, запрос данных первого запуска повторно |
| `404` | Не найдено | Датасет или пользователь не существует |
| `409` | Конфликт | Пользователь с таким именем уже существует |
| `500` | Ошибка сервера | Ошибка БД, ИИ не дал ответа, ошибка Python-кода |
| `503` | Сервис недоступен | Новости ещё не загружены |
